# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive problems.

If you discover a vulnerability, contact the maintainer privately first and include:

- a clear description of the issue
- affected files or features
- reproduction steps if available
- impact assessment

## Sensitive Data

Do not publish:

- HomeKit home names tied to private addresses
- RTSP camera URLs
- camera usernames or passwords
- access tokens, bridge endpoints, or local IP addresses

If a bug report includes sensitive local configuration, redact it before sharing.
